## 데이터 소비 방법
최종 사용자 - 데이터 소비자 - 데이터 생성자 - 원천 데이터 제공자
1. **요청-응답**  
소비자가 생성자에게 데이터를 요청하면, 생성자는 원천 데이터 제공자에게 다시 데이터를 요청하며, 소비자는 이렇게 응답받은 데이터를 최종 사용자에게
보여준다.
2. **메시지 전달**  
생성자가 주기적으로 원천 데이터 제공자에게 요청하여 수신한 데이터를 메시징 플랫폼에 전달하고, 소비자는 메시징 플랫폼을 이용하여 수신한 데이터를
내부 메모리 또는 인 메모리 DB에 저장하여 사용한다. 따라서 최종 사용자가 요청할 때마다 해당 데이터를 바로 보여준다.
3. **양방향 통신**  
최종 사용자와 생성자가 웹소켓을 이용하여 연결되어, 만약 생성자가 원천 데이터 제공자로부터 데이터를 수신하면 즉시 최종 사용자에게 해당 데이터를
전달한다.
